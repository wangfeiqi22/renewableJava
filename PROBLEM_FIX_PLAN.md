# 问题与修复计划清单

## 1. 司机角色功能理解偏差与实现缺失
### 现状分析
需求文档明确区分了 **车队司机 (Type A)** 和 **个人入驻司机 (Type B)**，但当前系统实现存在混淆和功能缺失：
- **车队司机 (Type A)**：
  - **需求**：接收车队派单，参与公海抢单（需授权），有固定的车辆管理。
  - **现状**：已实现基本的接收派单，但缺乏公海抢单功能和车队授权机制。
- **个人司机 (Type B)**：
  - **需求**：**禁止**抢公海订单，仅可**自建**清运单（自主接活录入），无联单打印功能。
  - **现状**：错误地为无车队司机提供了“抢单大厅”功能（`fetchPoolOrders`），这与需求严重冲突。个人司机应该是“自带业务进场”，而不是在平台上抢活。

### 修正计划
1.  **重构司机作业台 (`DriverTaskView.vue`)**：
    - 移除个人司机的“抢单大厅”入口。
    - 为个人司机新增“自主建单”入口（类似用户下单，但更简化，直接录入托运人信息）。
2.  **后端权限调整**：
    - `OrderController` 中限制个人司机调用抢单接口。
    - 新增个人司机专用建单接口。

## 2. 核心功能遗漏清单
以下功能在需求文档中明确规定，但当前代码完全缺失：

| 功能模块 | 需求描述 | 当前状态 | 修复优先级 |
| :--- | :--- | :--- | :--- |
| **AI 图像识别** | 用户/司机上传照片时，自动识别垃圾类型并估算重量/方量。 | ❌ 未实现 (纯手动选择) | High |
| **照片水印** | 所有业务照片必须强制添加时间戳、GPS经纬度、订单号水印。 | ❌ 未实现 | High |
| **电子联单** | 司机装车后需调用城管接口生成电子联单。 | ❌ 未实现 | Medium |
| **路径规划/轨迹** | 需实时记录车辆轨迹并在地图展示，提供导航。 | ❌ 未实现 | Medium |
| **异常上报** | 司机在作业过程中发现异常（如垃圾类型不符）的上报流程。 | ❌ 未实现 | Medium |
| **清运站分级** | 区分“清运站”和“中转站”，限制司机目的地选择。 | ⚠️ 仅有字段标识，无逻辑控制 | Medium |

## 3. 业务流程逻辑错误
1.  **个人司机抢单逻辑**：当前代码允许无车队司机抢单 (`!user.fleetId`)，需求文档明确禁止。个人司机应是“自主创建清运单”。
2.  **称重流程**：当前仅有清运站简单的“称重入库”，缺乏多次称重（皮重/毛重）和异常处理逻辑。

## 4. 下一步行动建议
1.  **立即修正**：禁用个人司机抢单功能，开发“自主建单”页面。
2.  **技术攻关**：引入水印处理库（前端 Canvas 或后端 Java 处理）和模拟 AI 识别服务（可先用 Mock 数据占位）。
3.  **完善字段**：数据库 `Order` 表需增加 `photo_url` (带水印), `gps_trace`, `station_type` 等字段。
