# 智慧垃圾清运系统开发计划 v4.1

## 1. 背景与目标
本开发计划依据《智慧垃圾清运系统PRD_v4.1》制定，覆盖系统从需求落地、架构搭建、核心功能实现、AI客服能力建设、第三方集成、测试验收到上线交付的全流程工作安排。

目标：
- 交付可上线运行的现代化 Web 应用系统（Spring Boot 3.0 + Vue 3 + PostgreSQL）
- 覆盖 PRD 核心功能模块（智能下单、调度、路线、监控、数据分析、站点分类、AI客服）
- 满足 PRD 非功能指标：稳定性 > 99.9%、客服界面响应 < 500ms、支持在线用户 > 1000、HTTPS 传输与数据安全

## 2. 范围说明

### 2.1 交付范围（MVP→正式版）
- **MVP（可演示/可验证）**：
  - 用户注册/登录与基础角色权限
  - AI客服对话：会话创建、消息收发、历史记录查询、基础 FAQ
  - 基础清运单：创建、状态流转、基础查询
  - 站点类型（清云站/中转站）与目的地选择限制（基础规则）
  - Docker Compose 一键部署（前后端+DB）
- **正式版（上线版）**：
  - 全量业务模块完善（调度、抢单池、轨迹/拍照留痕、监管驾驶舱、报表）
  - AI客服：意图识别、多轮对话、人机转接、知识库管理
  - 安全与合规：审计、加密、备份恢复、图片防篡改与校验
  - 完整测试与验收闭环

### 2.2 不在本期范围（需单独立项）
- 城管平台接口对接的具体协议联调细节（若外部接口未明确）
- 语音输入/输出在各端的完整适配（需产品/技术进一步确认终端形态与权限）
- 复杂算法级路线优化（如多车辆/多站点全局最优）若超出规则引擎能力

## 3. 技术架构与实施基线
- **后端**：Spring Boot 3.0，RESTful API，分层架构（Controller/Service/Repository），统一鉴权与审计
- **前端**：Vue 3 + Element Plus（或 Ant Design Vue），响应式 SPA
- **数据库**：PostgreSQL，按领域拆分表结构（用户、会话、对话消息、知识库、订单、站点、轨迹与图片索引等）
- **部署**：Dockerfile + docker-compose；生产建议增加 Nginx 反代与 HTTPS 终止
- **UI 风格**：绿色主色调（#00C851、#4CAF50），科技感与人性化交互

## 4. 里程碑与迭代计划（建议 6 个迭代）
说明：采用迭代交付，每个迭代输出可验证增量，优先满足 PRD 验收指标。

### 迭代 0：需求冻结与设计基线
- PRD 评审与范围冻结（MVP/正式版边界）
- 领域模型与数据库 ER 初版
- API 清单（OpenAPI/Swagger）与前后端契约确定
- UI 设计基线（绿色主题、对话界面规范、组件规范）

### 迭代 1：基础框架与用户体系
- 后端工程骨架（Spring Boot 3.0）、统一异常/日志、健康检查
- 用户注册/登录、角色权限（RBAC 最小集）
- 前端工程骨架（Vue 3）、路由、状态管理、基础布局与主题
- Docker Compose 端到端拉起（前后端+DB）

### 迭代 2：会话与AI客服 MVP（可演示）
- 会话管理：新建会话、会话列表、会话历史
- 对话消息：发送/接收、历史记录查询与分页
- UI：对话气泡、绿色主题、打字动画（typing）
- 性能基线：接口 P95 响应与前端交互延迟采集

### 迭代 3：知识库与问答策略
- 知识库管理：分类、条目、版本、上下线
- FAQ 匹配策略（关键词/向量检索可择一），可配置命中阈值
- AI 集成方案落地：
  - 方案 A：Rasa（自建）——意图识别、多轮对话
  - 方案 B：商业 API ——统一封装 Provider 接口，支持切换
- 人机转接：转接触发规则、会话标记、人工处理入口（管理端）

### 迭代 4：清运业务主链路（与AI客服联动）
- 清运单：创建、查询、状态流转（装车/进站/卸货/完成）
- 站点分类：清云站/中转站，目的地选择限制与权限差异
- 图片留痕：上传、EXIF 保留校验、哈希校验、关联订单/车次
- 监管要点沉淀：审计日志、关键行为追溯

### 迭代 5：集成、压测、上线准备
- 外部集成：城管平台联单（若接口到位）、地图服务（定位/轨迹展示）
- 安全加固：HTTPS、鉴权策略、敏感字段加密、权限矩阵复核
- 压测与容量评估：在线用户 > 1000、关键接口并发、数据库索引优化
- 发布与回滚：镜像版本化、配置分环境、备份与恢复演练

## 5. 模块工作分解（WBS）

### 5.1 后端（Spring Boot 3.0）
- 用户管理：注册、登录、角色、权限、审核（若启用）
- 会话管理：会话创建/关闭、会话状态、人机转接状态
- 对话管理：消息存储、检索、分页、全文搜索
- 知识库管理：条目 CRUD、分类、版本控制、上下线
- AI 接入层：Provider 抽象（Rasa/商业 API），超时与降级策略
- 安全与审计：JWT/Session、权限校验、操作日志、敏感信息脱敏

### 5.2 前端（Vue 3）
- 登录/注册：表单校验、错误提示、登录态保持
- 对话页：消息气泡、输入框、打字动画、已读状态、历史查询
- 管理页：知识库管理、人机转接处理、会话/消息检索
- 主题与组件规范：绿色主色（#00C851/#4CAF50）、统一间距与字体

### 5.3 数据与运维
- PostgreSQL 表结构与索引设计（用户、会话、消息、知识库、订单等）
- 加密存储策略（敏感字段、密钥管理方案）
- 备份策略：定期全量/增量、保留周期、恢复演练

### 5.4 测试与质量
- 单元测试：Service/Repository 核心逻辑覆盖
- 集成测试：关键 API 端到端（含 DB）
- E2E 测试（建议）：核心路径（登录→发起会话→对话→查询历史）
- 性能测试：接口 P95、前端交互延迟、并发 1000 用户场景

## 6. 关键非功能指标落地方案
- **< 500ms 响应**：接口缓存、索引优化、分页与限流、前端虚拟列表（如历史消息）
- **> 99.9% 稳定性**：健康检查、熔断降级（AI 调用）、数据库连接池与重试策略、可观测性
- **数据安全**：HTTPS、最小权限、敏感字段加密、审计日志与告警
- **高并发**：无状态服务水平扩展、数据库读写分离预留、异步化处理（图片/AI）

## 7. 验收对照（摘自 PRD）
- 功能：注册审核/权限生效、司机类型规则、拍照节点与 GPS/时间戳、站点限制、AI 问答与转接
- 性能：客服界面响应 < 500ms、在线用户 > 1000
- 安全：HTTPS、图片哈希校验、防篡改、加密与备份
- 技术：Spring Boot 3.0 + Vue 3 + PostgreSQL 部署成功，稳定性 > 99.9%

## 8. 风险与依赖
- 外部接口（城管平台、地图服务）不确定：提前确定协议与联调窗口，提供 Mock 服务并行开发
- AI 方案选择不明确：采用 Provider 抽象，支持 Rasa/商业 API 互换
- 性能指标挑战：提前建立性能基线与压测脚本，迭代内持续优化而非临上线突击
