# 垃圾分类拍照智能识别优化设计 (AI Waste Recognition Optimization)

## 1. 目标与背景
提升当前系统中“拍照智能识别”模块的准确率至95%以上，响应时间控制在2秒内。通过增强预处理、扩充边缘数据集、引入置信度验证与用户反馈闭环，并辅以A/B测试验证效果，打造全方位的智能垃圾分类模型与工程架构。

## 2. 核心架构与模型策略

### 2.1 算法模型设计 (四大类别)
* **类别体系**：可回收物、有害垃圾、湿垃圾、干垃圾。
* **模型选型**：推荐使用 **YOLOv8** 或 **EfficientNet-B4**，在轻量级与准确率之间取得平衡，结合 TensorRT 加速推理，确保端到端耗时 < 2s。
* **数据集扩充 (10万+样本)**：
  * **基础数据**：常规生活场景垃圾图片。
  * **边缘案例注入**：破损纸箱、混合形态（如带汤汁的塑料盒）、严重挤压变形的塑料瓶等。
  * **数据增强策略**：自动生成不同光照、部分遮挡、动态模糊等样本。

### 2.2 图像预处理增强 pipeline
在照片输入模型前，通过轻量级算法执行：
1. **自动白平衡 (Auto White Balance)**：修正暗光与背光。
2. **透视变换与畸变校正**：针对斜角拍摄的图片，利用边缘检测自动拉平。
3. **显著性目标检测 (Saliency Detection)**：对杂乱背景（如街道、草地）进行模糊或裁剪，仅保留前景垃圾主体。

## 3. 业务工程落地设计

### 3.1 多级置信度验证机制
当模型输出结果时，后端将携带 `confidence` (置信度得分 0-100%) 返回：
* **>= 85% (高置信)**：直接填充结果，展示详情报告。
* **< 85% (低置信/疑似模糊)**：
  * 前端弹出 **“人工复核/二次确认”** 弹窗，显示 AI 推荐的 Top 2 选项。
  * 用户需手动点击确认或更改。

### 3.2 识别报告与 UI 交互 (前端实现)
* **展示字段**：
  1. 物品推测名称 (如“矿泉水瓶”)
  2. 所属分类 (如“可回收物”)
  3. 处理建议 (如“请倒空液体后压扁投入”)
  4. 环保提示 (如“回收该物品可节约...”)
* **UI 组件**：设计一个类似卡片或弹窗的 `AiRecognitionReport` 组件。

### 3.3 用户纠错反馈闭环 (Data Flywheel)
* 若用户发现识别错误并手动修改了垃圾分类，前端将触发反馈接口：
  `POST /api/ai/feedback`
  **Payload**: `{ original_image_url, predicted_type, user_corrected_type, confidence }`
* 后端记录至 `ai_training_feedback` 库，作为后续迭代（主动学习）的金标准数据。

### 3.4 A/B 测试验证方案
* **实验组(B组)**：使用新版带预处理的增强模型与交互。
* **对照组(A组)**：使用旧版基础模型。
* **分流策略**：按 `userId` 的奇偶性或 Hash 取模进行 50/50 分流。
* **监测指标**：
  * 核心：一次性识别采纳率 (准确率) 是否提升 >= 20%。
  * 次要：平均识别耗时、用户手动修改频次、最终下单转化率。

## 4. 实施计划 (当前会话范围)

1. 更新前端 `OrderCreateView.vue` 及 `DriverTaskView.vue` 中的四大分类枚举。
2. 重构前端的 AI 分析逻辑（模拟预处理延迟、随机置信度、输出详细报告）。
3. 开发“低置信度复核弹窗”与“用户纠错反馈”UI组件。
4. 后端添加/模拟 A/B 测试分流与反馈收集接口 (Mock 层面)。
